MODULE 2.1 — INTRODUCTION TO REGULARIZATION (2.1_lesson.ipynb)
================================================================================

--- 2.1 Prompt 1: Overfitting Visualization ---

Create a Plotly visualization that shows three subplots side by side to
illustrate underfitting, good fit, and overfitting.

Generate 20 synthetic data points from y = 2x + 1 with Gaussian noise
(std=3, seed=42). Then fit three models:
- Underfitting: a horizontal line at the mean of y
- Good fit: a degree-1 polynomial (linear)
- Overfitting: a degree-15 polynomial

Plot the data as blue scatter points in all three panels, with the model
fit as a line (red for under/overfit, green for good fit). Title: "The
Fitting Spectrum: From Underfitting to Overfitting". Height 400px.


--- 2.1 Prompt 2: Bias-Variance Trade-off ---

Create a Plotly line chart showing the bias-variance trade-off.

Use model complexity from 0.1 to 3 (100 points). Define:
- Bias squared = 1 / complexity
- Variance = 0.3 * complexity^2
- Total error = bias^2 + variance + 0.1 (irreducible noise)

Plot all three curves (blue for bias, orange for variance, red for total
error). Add a dashed green vertical line at the optimal complexity (minimum
total error) with annotation "Optimal Complexity". Height 400px.


--- 2.1 Prompt 3: L1 vs L2 Geometry ---

Create a Plotly visualization with two subplots showing the geometric
constraint regions for L2 (Ridge) and L1 (Lasso) regularization.

- Left panel: L2 constraint — draw a filled circle (unit radius) in blue
  with 30% opacity. Title: "L2 (Ridge) Constraint"
- Right panel: L1 constraint — draw a filled diamond (unit radius) in
  orange with 30% opacity. Title: "L1 (Lasso) Constraint"

Add dashed gray axis lines to both panels. Label axes as beta_1 and
beta_2. Set axis range to [-1.5, 1.5]. Height 400px. No legend.

MODULE 2.2 — BUILD REGULARIZED MODELS (2.2_lesson.ipynb)
================================================================================

--- 2.2 Prompt 1: Load and Explore Training Data ---

Mount Google Drive and load training.csv from the path:
/content/drive/MyDrive/IR ML Cert/MLCert Course 3/Course 3 Data/

Print the shape of the data and the normalized value counts of the
SEM_3_STATUS column (our target variable).


--- 2.2 Prompt 2: Build Preprocessing Pipeline ---

Using scikit-learn's ColumnTransformer, create a preprocessing pipeline for
our student data with three transformers:

1. MinMaxScaler for bounded features: HS_GPA, GPA_1, GPA_2, DFW_RATE_1,
   DFW_RATE_2
2. StandardScaler for larger numeric features: UNITS_ATTEMPTED_1,
   UNITS_ATTEMPTED_2
3. OneHotEncoder for categorical features: GENDER, RACE_ETHNICITY,
   FIRST_GEN_STATUS — drop the categories 'Female', 'Other', 'Unknown'
   respectively. Use handle_unknown='ignore' and sparse_output=False.

Set remainder='drop' to exclude any other columns.


--- 2.2 Prompt 3: Build L2 (Ridge) Logistic Regression Pipeline ---

Create a scikit-learn Pipeline with two steps:
1. The preprocessor we just defined
2. A LogisticRegression with L2 penalty, C=1.0, class_weight='balanced',
   solver='lbfgs', max_iter=1000, random_state=42

Name the steps 'preprocessing' and 'classifier'. Print the model and
display the pipeline.


--- 2.2 Prompt 4: Build L1 (Lasso) Logistic Regression Pipeline ---

Create the same pipeline but with L1 penalty instead of L2. Use
solver='saga' (required for L1). Keep C=1.0, class_weight='balanced',
max_iter=1000, random_state=42.


--- 2.2 Prompt 5: Build ElasticNet Logistic Regression Pipeline ---

Create the same pipeline with ElasticNet penalty. Use solver='saga',
l1_ratio=0.5 (equal mix of L1 and L2), C=1.0, class_weight='balanced',
max_iter=1000, random_state=42.


--- 2.2 Prompt 6: Compare and Save All Models ---

Create a dictionary of all three models (L2 Ridge, L1 Lasso, ElasticNet).
Loop through them and print each model's penalty, C value, solver, and
l1_ratio (if applicable).

Then save each model as a pickle file to the models directory with
descriptive filenames like l2_ridge_logistic_model.pkl.

Mount Google Drive. Load training.csv from:

Then load four pickled model pipelines from the same directory:
- l2_ridge_logistic_model.pkl

- elasticnet_logistic_model.pkl

--- 2.3 Prompt 2: Train All Models ---

Loop through the dictionary of models and call .fit(X_train, y_train) on

training DataFrame and y_train is the SEM_3_STATUS column.


--- 2.3 Prompt 3: Compare Coefficients Across Models ---

For each trained model, extract the classifier coefficients and the
feature names from the preprocessor (using get_feature_names_out()).
Clean up the feature names by removing the transformer prefix.

Create a Plotly grouped bar chart comparing coefficient values across all
four models (Baseline, L2, L1, ElasticNet) for each feature. Use
barmode='group', height 500px. Rotate x-axis labels 45 degrees.


--- 2.3 Prompt 4: Coefficient Norms ---

For each model, compute:
- L2 norm: square root of sum of squared coefficients
- L1 norm: sum of absolute coefficient values

Display as a DataFrame. This measures overall model complexity — we expect
regularized models to have smaller norms than the baseline.


--- 2.3 Prompt 5: L1 Feature Selection Analysis ---

Then create a bar chart with green bars for selected features and light
gray bars for eliminated features. Title: "L1 (Lasso) Coefficients:
Green = Selected, Gray = Eliminated".


--- 2.3 Prompt 6: Cross-Validation Comparison ---

Run cross_val_score for all four models on all three metrics. Collect
results in a DataFrame showing mean and std for each metric per model.

Then create a Plotly subplot with three panels (F1, Precision, Recall)
showing bar charts with error bars for each model. Height 400px.


--- 2.3 Prompt 7: Save Trained Models ---

MODULE 2.4 — TUNE HYPERPARAMETERS (2.4_lesson.ipynb)

--- 2.4 Prompt 1: Load Data and Models for Tuning ---

Mount Google Drive. Load both training.csv and testing.csv from:
/content/drive/MyDrive/IR ML Cert/MLCert Course 3/Course 3 Data/

Create X_train, y_train from training data and X_test, y_test from test
data. The target is SEM_3_STATUS.

Load the three pickled model pipelines: L2, L1, and ElasticNet.


--- 2.4 Prompt 2: Tune L2 (Ridge) with GridSearchCV ---

Set up GridSearchCV to tune the L2 model's C parameter. Search over
C = [0.001, 0.01, 0.1, 1.0, 10.0, 100.0].

Use 5-fold StratifiedKFold (shuffle=True, random_state=42). Optimize
for F1 score on the minority class (pos_label='N'). Use n_jobs=-1 and
return_train_score=True.

Note: the parameter path in the pipeline is 'classifier__C'.

Fit the grid search on X_train and y_train. Print the best C value and
best CV F1 score. Then display a results table showing C, Train F1,
CV F1, CV F1 Std, and Rank for all C values.


--- 2.4 Prompt 3: Tune L1 (Lasso) with GridSearchCV ---

Do the same GridSearchCV for the L1 model over the same C values.
Print the best C and best CV F1 score. Display the results table.


--- 2.4 Prompt 4: Tune ElasticNet with 2D GridSearchCV ---

Tune the ElasticNet model over TWO hyperparameters simultaneously:
- classifier__C: [0.001, 0.01, 0.1, 1.0, 10.0, 100.0]
- classifier__l1_ratio: [0.1, 0.3, 0.5, 0.7, 0.9]

This gives 30 total combinations. Use the same CV and scoring setup.
Print the best C, best l1_ratio, and best CV F1 score. Display the
top 10 results sorted by rank.


--- 2.4 Prompt 5: Visualize ElasticNet Hyperparameter Grid ---

Create a heatmap of the ElasticNet GridSearch results using Plotly's
px.imshow. The x-axis is C values, y-axis is l1_ratio values, and the
color shows CV F1 Score. Use the Viridis color scale. This helps us see
which region of the hyperparameter space performs best.


--- 2.4 Prompt 6: Compare All Tuned Models ---

Create a comparison table showing the best model from each regularization
type with columns: Model, Best C, Best l1_ratio, CV F1 Mean, CV F1 Std.

Then create a bar chart comparing CV F1 scores with error bars across the
three tuned models. Use different colors for each model.


--- 2.4 Prompt 7: Select Best Model and Evaluate on Test Set ---

Identify the model with the highest CV F1 score. Extract its best
estimator from the GridSearchCV object.

Generate predictions on the test set. Calculate F1, Precision, Recall
(pos_label='N'), and Accuracy. Print a formatted summary.

Then print the full classification_report from scikit-learn.

Important: we only evaluate on the test set ONCE after all tuning is
complete to avoid data leakage.


--- 2.4 Prompt 8: Confusion Matrix ---

Create an interactive confusion matrix heatmap using Plotly for the best
model's test set predictions. Labels should be 'Left (N)' and 'Stayed (Y)'
for both axes. Use the Blues color scale with text annotations showing
counts. Height 400px.

Then print an interpretation: True Positives, True Negatives, False
Positives, and False Negatives with their counts.


--- 2.4 Prompt 9: Visualize C Effect on Performance ---

Create a 3-panel Plotly subplot showing how CV F1 score changes with C
(on a log10 scale) for each regularization type: L2, L1, and ElasticNet.
Include error bars. For ElasticNet, use the best l1_ratio at each C value.

This visualization shows the bias-variance trade-off in action —
small C (strong regularization) may underfit, large C (weak regularization)
may overfit.


--- 2.4 Prompt 10: Save Best Model ---

Save the best tuned model as a pickle file. Also save a dictionary
containing all three GridSearchCV objects and the best model name for
